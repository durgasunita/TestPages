Xray Test Plan[PG1] [DP2] 

Before You Begin

The Xray Test Plan [PG3] [DP4] (synonymous with Validation /Verification Plan) is an overall release test plan. Its purpose is to identify the Tests you want to run for a release, to schedule executions for them and then group the respective results.

Test Plan can be viewed as an elevated entity above Test Executions. It enables teams to get a consolidated overview of Tests that are associated, plan and group multiple test executions by digitally selecting testcases from test sets and, keep track of test results and overall test plan execution status throughout the project release cycles.

It is a good practice to develop Release Test Plan (Issue Type: Release Test Plan) and overall Test Plan (Issue Type: Test Plan) first before creating Test Set and Test Cases to support the Test Plan.  By using this approach, you can immediately associate a Test Set to its Test Plan and related Requirements, instead of updating the Test Plan at a later point in time and having to remember which Test Sets to assign to it.

Procedure to create Test Plan

To create a new Test Plan Issue, 

Step 1: Click the Create Issue at the top of the screen to open the Create Issue dialog box / page.

Step 2: Select the Project and on Issue Type select Test Plan.

Step 3: Under the General tab, type a Summary for the test execution, Fix Version and fill all appropriate fields mandatory fields are marked with an asterisk. You should also fill out the Fix Version to indicate that you're tracking the results of the Tests for that version. This will be used to automatically fill out the Fix Version of the planned Test Executions (i.e., the ones created from this Test Plan).

Step 4: Under the Test Plan Details tab, you can populate the Test Plan custom fields provided by Xray, namely the Begin/End date. In this tab, you can also add tests to the Test Plan.

[PG5] [DP6] 

Step 5: When you are satisfied with the content of your Test Plan, click the Create button.

[PG7] [DP8] The Test Plan is created and is in "Open" status.

Create Empty Test Sets

You can create an empty Test Set[PG9] [DP10]  whenever there is a need to work on new feature and testcases are not ready during planning phase.

Note: An empty Test Set will be not displayed in the Test Plan, ensure to add a dummy Test with Summary to reflect test objective of the feature.

 

Step 1: Click the Create Issue at the top of the screen to open the Create Issue dialog / page.

Step 2: Select the Project and on Issue Type, select Test Set.

Step 3: Fill out the General Tab fields Summary, Description, Test Group Type and Fix Version/s

Step 4: Click the Create button.

Add/Import Test Set/s to Test Plan 

Aside from adding Test Set/s to the Test Plan using the issue create screen, you can also do it in the Test Set issue page or directly in the tests section of the Test Plan issue page, as follows.

Step 1: Click Add.

Step 2: Click the Test from Test Sets... option to open the Issue Picker Dialog for adding the Tests that belong to that Test Set/s.

Step 3: Open the Select tab.

This tab provides you an issue selector with a suggested list of recent issues or you can search by the text contained in the Summary.

How to add issues using the Select tab:

- Input the desired Test Issue Key on the Tests field.
- Click on the Down Arrow on the Tests field and select the Test from its History Search list.
- Click on the + button which will prompt an Issue Selector Pop-Up. 

Step 4: Select Test Sets.[PG11] [DP12] 

Step 5:  Add [PG13] [DP14] Tests of the selected Test Sets by pressing "Add selected[PG15] [DP16] ".[PG17] [DP18]

Since the table only shows Test Runs for tests that are in the Test Plan and for Test Executions that are associated with the Test Plan, it's necessary to associate Test Executions with the Test Plan.[PG22] [DP23] 

You can also add existing Test Execution issues directly to a Test Plan. 

The selected Test Executions will be added, including the Tests associated with those Test Executions.

 

Create Subsequent Test Executions from the Test Plan

On completion of first interval of Test Execution, subsequent executions can be planned in the next interval based on

         Test Status, if there are any failed or [PG24] [DP25] blocked or skipped Tests from your previous intervals.

         Change in Test Set, due to addition or update of a Test Case based on changes to design and feature implementations

You will be required to plan and perform subsequent test executions or re-executions of specific Tests when there is a fix to the System/Product under test.

There are several ways to create Test Executions directly in the Test Plan issue page.

The most common and straightforward option to create a Test Execution for all the Tests, or for a subset of those that are currently in some consolidated status, is by using the Create Test Execution available in the Test Plan issue screen.

The Create Test Execution button will open a dropdown with some options that will allow you to select the Tests that will be part of the Test Execution. 

Please note

In order to create a related Test Execution from the Test Plan, the user must be allowed to the edit the current Test Plan since the results on the Test Execution will affect the consolidated information shown at the Test Plan. The user will only be able to create Test Executions in certain projects, therefore the list of projects shown in the creation dialog will be filtered out accordingly.
